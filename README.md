# Ground Rule
## Scrum
* 오전 10시 (행 아웃)
* 내용
  * 컨디션
  * 구현 상황
  * 오늘의 목표

## Convention
* 커밋 메세지
  * Feat: 새로운 기능 추가
  * Fix: 버그 수정
  * Docs: 문서 수정
  * Style: 코드 포맷팅 (코드 변경이 없는 경우)
  * Test: 테스트 코드
  * Chore: 빌드 업무, 패키지 매니저 수정
  * [참고](https://doublesprogramming.tistory.com/256)

* Git issue 활용
  * 커밋 메세지로 이슈 닫기
  * `Fixed #n`

* 브랜치 구분
  * master: 최종 버전
  * dev: 기능 개발 완료 후 dev 브랜치에 병합
  * 클래스/Feature/기능: 기능 개발 브랜치
